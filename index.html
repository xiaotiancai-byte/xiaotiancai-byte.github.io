AOP术语

Posted on 2019-12-19

AOP术语

  1. target：目标类，需要被代理的类。例如：UserService
  2. JointPoint（连接点）：所谓连接点是指那些可能被拦截到的方法。例如：所有的方法
  3. PointCut 切入点：已经被增强的连接点。例如：addUser()
  4. advice 通知/增强，增强代码。例如：after、before
  5. Weaving（织入）：是指把增强advice应用到目标对象target来创建新的代理类对象proxy的过程
  6. proxy代理类
  7. Aspect（切面）：是切入点pointcut和通知advice的结合
    • 一条线是一个特殊的面
    • 一个切入点(PointCut)和一个通知(advice)能组成一个特殊的面

git版本控制

Posted on 2019-12-05

git版本日志

  • git log:显示完整的日志信息
  • git log --pretty=oneline:显示完整的哈希索引值和头指针
  • git reflog:显示部分哈希索引值，索引编号和头指针
  • git log --oneline:显示部分哈希索引值和头指针，只显示当前版本和历史版本

git版本控制

  • 基于索引值操作：git reset --hard [索引哈希值]
  • 使用^符号：git reset --hard HEAD^ hint:一个^只能后退一步，n个表示后退n步
  • 使用~符号：git reset --hard HEAD~n hint:表示后退n步

约束文档详解

Posted on 2019-12-02 
*约束：规定xml的书写规则
    *作为框架的使用者
        1.能够在xml中引入约束文档
        2.能够读懂约束文档
    *分类：
        1.DTD：一种简单的约束技术
        2.Schema：一种复杂的约束技术    
    *DTD：
        *引入dtd文档到xml文档中
            *内部dtd：将约束规则定义在xml文档中
            *外部dtd：将约束规则定义在外部dtd文件中
                *本地：<!DOCTYPE 根标签名 SYSTEM "dtd文件位置">
                *网络：<!DOCTYPE 根标签名 PUBLIC "dtd文件名字" "dtd文件位置URL">
    *Schema:
        *<?xml version="1.0" encoding="UTF-8"?>
xmlns(xmlNameSpace)：添加命名空间   后面不跟命名时表示默认自动
xmlns:context(使用"http://www.springframework.org/schema/context"时需要加前缀命名空间context:)

更改Hexo博客主题的具体步骤-1

Posted on 2019-12-01 Edited on 2019-12-02

  1. 首先进入https://hexo.io/themes 挑选一个自己喜欢的主题点击进入到该博客

  2. 点击github小图标或链接进入博主的github

  3. 进入到github后打开博主的仓库

  4. 搜索主题的名字

  5. 点击复制主题链接地址

  6. 到博客根目录下右键git bash ， 下载主题git clone https://github.com/yelog/hexo-theme-3-hexo.git themes/hexo3

  7. 再打开博客根目录下的配置文件

  8. 修改theme的属性值为本地主题文件夹的名字

9.最后保存 git clean、git g、git s、git d就可以啦:slightly_smiling_face:
